## 摘要

本研究以RPA（机器人流程自动化）技术在金融领域的应用为研究对象，通过问卷调查收集500份有效样本，综合运用描述性统计、信效度检验、结构方程模型、时间序列分析、文本分析、DEMATEL-ISM和模糊AHP-TOPSIS等多元方法，系统分析了RPA技术应用的影响因素、作用机制和发展策略。

研究发现：(1)任务技术匹配(TTF)、社会影响(SI)显著正向影响感知价值(PV)；(2)感知价值(PV)显著正向影响使用意愿(BI)，风险感知(RD)显著负向影响使用意愿；(3)数据孤岛、AI融合困难、系统兼容性是影响RPA应用的核心原因因素；(4)LLM+RAG是最具优先发展价值的技术方向。本研究为金融机构RPA技术应用提供了理论依据和实践指导。

**关键词**: RPA技术；金融领域；影响因素；结构方程模型；多方法融合

## 第一章 研究背景与问题提出

### 1.1 研究背景

随着数字经济的快速发展，金融机构面临着效率提升、成本控制、合规管理等多重挑战。RPA（Robotic Process Automation，机器人流程自动化）作为一种能够模拟人类操作、自动执行规则化任务的技术，正在金融领域得到广泛应用。从账户管理到交易处理，从合规报告到客户服务，RPA技术正在重塑金融机构的运营模式。

然而，RPA技术在金融领域的应用仍面临诸多挑战。技术成熟度、数据安全、人才短缺、系统集成等问题制约着RPA的深度应用。如何系统识别RPA技术应用的影响因素，揭示其作用机制，制定有效的发展策略，成为金融机构数字化转型的重要课题。

### 1.2 研究问题

本研究围绕以下核心问题展开：

1. **影响因素识别**: 影响金融机构RPA技术应用的关键因素有哪些？
2. **作用机制分析**: 各影响因素之间如何相互作用？其影响路径是什么？
3. **发展策略制定**: 如何基于影响因素和作用机制，制定有效的RPA发展策略？

### 1.3 研究意义

**理论意义**: 本研究整合TTF理论、社会影响理论、感知价值理论等，构建RPA技术应用影响因素模型，丰富了技术创新采纳理论在金融科技领域的研究。

**实践意义**: 研究结论为金融机构RPA技术应用提供决策参考，为政策制定者完善金融科技政策提供依据，为RPA厂商优化产品设计提供指导。

#### 3.3.3 关键发现

1. **TTF对PV的影响最强** (β=0.270)
   - 任务技术匹配是感知价值的核心驱动因素
   - 技术与业务需求的匹配度越高，用户感知价值越大

2. **PV对BI的影响显著** (β=0.219)
   - 感知价值是使用意愿的关键影响因素
   - 提升用户价值感知可有效促进RPA应用

3. **RD对BI的负向影响** (β=-0.098)
   - 风险感知抑制使用意愿
   - 降低风险感知是促进应用的重要途径

4. **SI调节效应不显著**
   - 社会影响对TTF-PV关系的调节作用未得到验证
   - 可能需要考虑其他调节变量

## 第四章 研究结论与讨论

### 4.1 主要研究结论

#### 4.1.1 影响因素识别

本研究识别出影响RPA技术在金融领域应用的六大核心因素：

| 因素类型 | 因素 | 影响机制 |
|----------|------|----------|
| 技术因素 | 任务技术匹配 | 正向影响感知价值 |
| 技术因素 | 系统兼容性 | 原因因素，影响成本和风险 |
| 技术因素 | AI融合能力 | 原因因素，影响智能化水平 |
| 环境因素 | 社会影响 | 正向影响感知价值 |
| 环境因素 | 合规风险 | 结果因素，受技术因素影响 |
| 组织因素 | 人才储备 | 结果因素，受技术因素影响 |

### 4.2 理论贡献

1. **整合多理论视角**: 本研究整合TTF理论、社会影响理论、感知价值理论，构建了RPA技术应用影响因素的综合模型，丰富了技术创新采纳理论。

2. **揭示作用机制**: 通过SEM分析，揭示了TTF→PV→BI和SI→PV→BI的影响路径，以及RD对BI的直接负向影响，深化了对RPA技术应用机制的理解。

3. **多方法融合创新**: 综合运用SEM、DEMATEL-ISM、TOPSIS等方法，实现了影响因素识别、作用机制分析、发展策略制定的有机统一。

### 4.3 实践启示

#### 4.3.1 对金融机构的启示

1. **提升任务技术匹配度**
   - 深入分析业务需求，选择适配的RPA技术
   - 加强技术与业务的融合，提升感知价值

2. **降低风险感知**
   - 完善数据安全措施，建立合规管理体系
   - 加强风险沟通，提升用户信任

3. **把握技术发展趋势**
   - 重点关注LLM+RAG技术方向
   - 适时引入智能化RPA解决方案

#### 4.3.2 对政策制定者的启示

1. **完善政策体系**
   - 出台RPA应用指导规范
   - 建立数据安全与合规标准

2. **支持技术创新**
   - 加大对LLM+RPA等前沿技术的支持
   - 推动产学研协同创新

3. **加强人才培养**
   - 完善RPA人才培养体系
   - 推动认证制度建设

#### 4.3.3 对RPA厂商的启示

1. **优化产品设计**
   - 提升系统兼容性
   - 加强AI融合能力

2. **降低实施门槛**
   - 开发低代码平台
   - 提供完善的培训支持

3. **强化合规能力**
   - 内置合规检查功能
   - 支持审计追溯

### 4.4 研究局限与展望

#### 4.4.1 研究局限

1. **样本代表性**: 样本主要来自金融行业，对其他行业的适用性有待验证。

2. **横截面数据**: 本研究采用横截面数据，无法揭示影响因素的动态变化。

3. **调节效应**: SI的调节效应未得到验证，可能存在其他调节变量。

#### 4.4.2 未来研究展望

1. **纵向研究**: 开展纵向追踪研究，揭示影响因素的动态演变。

2. **跨行业比较**: 扩大样本范围，进行跨行业比较研究。

3. **调节机制**: 探索其他可能的调节变量，如组织支持、技术准备度等。

4. **案例研究**: 结合典型案例，深入分析RPA应用的成功因素。
