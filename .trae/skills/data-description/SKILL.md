## 1. survey_data_simulated.csv (问卷模拟数据)

### 基本信息
- **记录数**: 500条
- **字段数**: 360个变量
- **数据来源**: 基于《完整版问卷.md》生成的模拟数据

### 核心字段结构

#### 基础信息 (Q1-Q2)
| 字段名 | 说明 | 取值范围 |
|--------|------|----------|
| respondent_id | 受访者ID | R0001-R0500 |
| submit_time | 提交时间 | 2026-02-10 ~ 2026-02-20 |
| Q1_industry | 所属行业 | 银行业/证券业/保险业/基金业等12类 |
| Q1_department | 所属部门 | 零售银行部/投资部/理赔部等 |
| Q2_employee_scale | 企业规模 | 1-4 (小型→超大型) |
| Q2_enterprise_type | 企业类型 | 1-4 (国企/民企/外企/混合) |
| Q2_digital_stage | 数字化阶段 | 1-4 (起步→成熟) |
| Q2_rpa_years | RPA使用年限 | 1-4 (1年内→5年以上) |
| Q2_rpa_scale | RPA部署规模 | 1-3 (小→大) |
| Q3_eco_role | 生态角色 | 供应端/应用端/环境端等6类 |

#### RPA产品使用 (Q4-Q5)
| 字段名 | 说明 |
|--------|------|
| Q4_products | 使用的RPA产品 (多选) |
| Q5_1 ~ Q5_22 | 各产品使用频率 (1-5分) |

#### 技术认知与使用 (Q6-Q7)
| 字段名 | 说明 |
|--------|------|
| Q6_1 ~ Q6_20 | 技术认知程度 (1-5分) |
| Q7_1_2 ~ Q7_6_5 | 技术组合使用情况 (二元矩阵) |

#### 技术重要性排序 (Q8)
| 字段名 | 说明 |
|--------|------|
| Q8_1_rank ~ Q8_12_rank | 12项技术重要性排序 (1-12) |

#### 核心量表 (Q9-Q14) - SEM分析核心变量

**任务-技术匹配 (TTF) - Q9**
| 字段 | 题项 | 取值 |
|------|------|------|
| Q9_TTF1 | 技术功能匹配任务需求 | 1-5 |
| Q9_TTF2 | 技术易于使用 | 1-5 |
| Q9_TTF3 | 技术可靠稳定 | 1-5 |
| Q9_TTF4 | 技术学习成本低 | 1-5 |

**社会影响 (SI) - Q10**
| 字段 | 题项 | 取值 |
|------|------|------|
| Q10_SI1 | 领导支持RPA应用 | 1-5 |
| Q10_SI2 | 同事认可RPA价值 | 1-5 |
| Q10_SI3 | 行业趋势推动 | 1-5 |
| Q10_SI4 | 政策引导支持 | 1-5 |

**感知价值 (PV) - Q11**
| 字段 | 题项 | 取值 |
|------|------|------|
| Q11_PV1 | 提升工作效率 | 1-5 |
| Q11_PV2 | 降低运营成本 | 1-5 |
| Q11_PV3 | 提升工作质量 | 1-5 |
| Q11_PV4 | 增强竞争力 | 1-5 |

**风险感知 (RD) - Q12**
| 字段 | 题项 | 取值 |
|------|------|------|
| Q12_RD1 ~ Q12_RD8 | 技术/安全/合规/成本等风险 | 1-5 |

**收益感知 (RV) - Q13**
| 字段 | 题项 | 取值 |
|------|------|------|
| Q13_RV1 ~ Q13_RV6 | 效率/成本/质量/创新等收益 | 1-5 |

**技术成熟度评估 (Q14) - 模糊三角数**
| 字段模式 | 说明 |
|----------|------|
| Q14_X_low | 悲观估计值 |
| Q14_X_mid | 最可能值 |
| Q14_X_high | 乐观估计值 |

X取值范围: 1-26，覆盖26项技术的成熟度评估

#### 技术组合偏好 (Q15)
| 字段模式 | 说明 |
|----------|------|
| Q15_X_A | 技术A选择 |
| Q15_X_A_val | 技术A偏好值 |
| Q15_X_B | 技术B选择 |
| Q15_X_B_val | 技术B偏好值 |

X取值范围: 1-45，覆盖45组技术对比

#### 应用场景与需求 (Q16-Q18)
| 字段 | 说明 |
|------|------|
| Q16_scenarios | 应用场景 (多选) |
| Q17_future_expectations | 未来期望 (多选) |
| Q18_talent_needs | 人才需求 (多选) |

#### 开放题 (Q19-Q20)
| 字段 | 说明 |
|------|------|
| Q19_pain_points | 应用痛点 (开放文本) |
| Q20_suggestions | 发展建议 (开放文本) |

## 分析建议

### 1. 结构方程模型 (SEM)
- 使用Q9-Q13量表数据
- 构建TTF→PV→使用意愿模型
- 验证SI的调节作用

### 2. 时间序列分析
- tech_timeline.csv: 技术发展阶段分析
- policy_timeline.csv: 政策演进趋势分析
- 技术与政策协同效应分析

### 3. 文本分析
- Q19/Q20开放题: 痛点与建议挖掘
- 使用BERT/BERTopic进行主题建模
- 情感分析与关键词提取

### 4. 多方法融合
- DEMATEL-ISM: 因素关联分析
- 模糊AHP-TOPSIS: 技术成熟度评估
- 贝叶斯网络: 风险传导分析

## 注意事项

1. **数据质量**: 模拟数据基于问卷逻辑生成，实际调研需验证
2. **缺失值处理**: 部分字段存在条件跳转，需按问卷逻辑处理
3. **编码转换**: 分类变量已编码，分析时需参考编码说明
4. **时间格式**: 时间字段采用YYYY-MM格式，精确到月，便于时间序列分析
