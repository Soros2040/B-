## 二、核心方法详细说明

### 方法1：描述性统计分析

**用途**：RQ1现状识别的基础分析

**分析内容**：
- 样本特征分布（行业、规模、RPA使用情况等）
- 痛点/需求评分的均值、标准差、分布
- 行业间差异的显著性检验（ANOVA、Kruskal-Wallis）

**数据需求**：
```
问卷数据：
├── 基本信息：行业、规模、企业性质、数字化转型阶段
├── RPA使用情况：使用年限、部署规模、应用场景
├── 痛点评分：Likert 5点量表（15个痛点项）
└── 需求评分：Likert 5点量表（12个需求项）

样本量要求：≥200份
```

**输出成果**：
- 样本特征分布表
- 痛点/需求评分统计表
- 行业差异检验表
- 可视化图表：柱状图、饼图、箱线图

**实施难度**：⭐（简单）

### 方法2：文本分析（TF-IDF + 主题建模）

**用途**：RQ1痛点识别的深度分析

**技术路线**：
```
Step 1: 文本预处理
        ├── 中文分词（jieba + 金融RPA自定义词典）
        ├── 去停用词
        └── 词性标注

Step 2: 关键词提取
        ├── TF-IDF权重计算
        ├── 关键词共现分析
        └── 关键词网络构建

Step 3: 主题建模
        ├── 方案A：BERTopic（需网络环境）
        └── 方案B：LDA + KMeans（离线可用）

Step 4: 结果可视化
        ├── 词云图
        ├── 主题分布图
        └── 关键词网络图
```

**数据需求**：
```
文本数据：
├── 问卷开放题答案（痛点描述、需求描述）
├── 专家访谈记录
├── 行业报告/案例文档
└── 政策文件

文本量要求：≥500条有效文本（每条≥20字）
```

**输出成果**：
- 痛点关键词清单
- 主题分布表
- 关键词共现网络图

**实施难度**：⭐⭐⭐（中等）

### 方法3：SEM结构方程模型

**用途**：RQ2因果分析的核心方法

**理论模型**（基于TOE框架）：
```
┌─────────────────────────────────────────────────────────────┐
│                    RPA需求影响因素模型                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【技术因素】                  【组织因素】                   │
│  ├── TF1: 技术成熟度          ├── OF1: 数字化程度            │
│  ├── TF2: 技术兼容性          ├── OF2: 管理层支持            │
│  └── TF3: AI融合程度          └── OF3: 实施团队能力          │
│         │                           │                        │
│         └───────────┬───────────────┘                        │
│                     │                                        │
│                     ▼                                        │
│              ┌─────────────┐                                │
│              │  RPA需求    │                                │
│              └─────────────┘                                │
│                     ▲                                        │
│         ┌───────────┴───────────────┐                        │
│         │                           │                        │
│  【环境因素】                  【痛点程度】                   │
│  ├── EF1: 政策支持            ├── PP1: 数据痛点              │
│  ├── EF2: 监管合规            ├── PP2: 技术痛点              │
│  └── EF3: 竞争压力            └── PP3: 人才痛点              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**数据需求**：
```
问卷数据（潜变量测量）：
每个潜变量至少3个观测变量，共15个观测变量
样本量要求：≥200份（建议10倍于观测变量数）
```

**实现工具**：semopy（Python）

**输出成果**：
- SEM路径图（含标准化系数）
- 模型拟合指标表（χ²/df、RMSEA、CFI、TLI等）
- 假设检验结果表
- 直接效应、间接效应、总效应表

**实施难度**：⭐⭐⭐⭐（较难）

**注意事项**：
- 若样本量不足，可使用PLS-SEM替代
- 需进行信效度检验（Cronbach's α > 0.7, AVE > 0.5）

### 方法4：模糊DEMATEL-ISM（★优化项）

**用途**：RQ2因果分析的辅助方法（用于汇报展示）

**优化说明**：
- 原方案：普通DEMATEL
- 新方案：**模糊DEMATEL**（使用pyDecision库）
- 优势：处理专家判断的不确定性，结果更稳健

**技术路线**：
```
Step 1: 构建模糊判断矩阵
        ├── 使用三角模糊数表示专家判断
        └── 多专家判断聚合

Step 2: DEMATEL计算
        ├── 计算综合影响矩阵
        ├── 计算影响度R和被影响度C
        ├── 计算中心度D和原因度P
        └── 绘制因果四象限图

Step 3: ISM层次划分
        ├── 确定阈值θ
        ├── 构建可达矩阵
        └── 层次划分

Step 4: 结果可视化
        ├── 因果四象限图
        └── 层次结构图
```

**数据需求**：
```
专家评分数据：
├── 专家数量：5-10人
├── 评分方式：模糊语言变量（无影响/弱/中/强）
├── 评分对象：痛点因素矩阵（5-6个因素）
└── 转换规则：语言变量 → 三角模糊数
```

**实现工具**：pyDecision库
```python
from pyDecision.algorithm import fuzzy_dematel_method
```

**输出成果**：
- 模糊影响矩阵
- 因果四象限图（中心度-原因度）
- 层次结构图（ISM分层）

**实施难度**：⭐⭐⭐（中等）

### 方法5：模糊AHP

**用途**：RQ3方案评估的权重计算

**技术路线**：
```
Step 1: 构建模糊判断矩阵
        ├── 使用三角模糊数表示专家判断
        └── Saaty 1-9标度转换为模糊数

Step 2: 计算模糊权重
        ├── 计算模糊几何平均值
        ├── 计算模糊权重
        └── 去模糊化（重心法）

Step 3: 一致性检验
        ├── 计算一致性指标CI
        ├── 计算一致性比率CR
        └── 要求CR < 0.1
```

**数据需求**：
```
AHP两两比较数据：
├── 专家数量：5-10人
├── 比较对象：技术方案（LLM+RAG、多模态AI等）
├── 评分标准：Saaty 1-9标度
└── 一致性要求：CR < 0.1
```

**实现工具**：pyDecision库
```python
from pyDecision.algorithm import fuzzy_ahp_method
```

**输出成果**：
- 模糊AHP权重表
- 一致性检验结果

**实施难度**：⭐⭐⭐（中等）

### 方法6：TOPSIS + VIKOR组合（★优化项）

**用途**：RQ3方案评估的综合排序

**优化说明**：
- 原方案：仅TOPSIS
- 新方案：**TOPSIS + VIKOR组合**
- 优势：TOPSIS关注距离理想解，VIKOR考虑群体效用和个体遗憾，两者互补

**TOPSIS方法**：
```
Step 1: 构建决策矩阵
Step 2: 规范化处理
Step 3: 加权规范化矩阵（使用模糊AHP权重）
Step 4: 计算正负理想解
Step 5: 计算相对贴近度
Step 6: 方案排序
```

**VIKOR方法**：
```
Step 1: 构建决策矩阵
Step 2: 确定最佳值和最差值
Step 3: 计算群体效用值S
Step 4: 计算个体遗憾值R
Step 5: 计算折衷评价值Q
Step 6: 方案排序
```

**组合分析**：
```
┌─────────────────────────────────────────────────────────────┐
│                    TOPSIS + VIKOR 组合分析                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  TOPSIS排序  ──────┐                                        │
│                    ├─────▶ 排序一致性检验                     │
│  VIKOR排序   ──────┘         │                              │
│                              │                              │
│                              ▼                              │
│                    ┌─────────────────┐                      │
│                    │  Spearman相关   │                      │
│                    │  ρ > 0.7 为一致  │                      │
│                    └─────────────────┘                      │
│                              │                              │
│                              ▼                              │
│                    ┌─────────────────┐                      │
│                    │  综合排序结果    │                      │
│                    │  （Borda计数法） │                      │
│                    └─────────────────┘                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**实现工具**：pyDecision库

**输出成果**：
- TOPSIS排序结果表

## 三、三角验证机制

### 3.1 验证维度

| 维度 | 说明 | 具体指标 |
|------|------|----------|
| **方法验证** | 不同方法对同一问题的结论一致性 | 重合度、相关性、方向一致性 |
| **数据验证** | 不同数据来源的结论一致性 | 问卷vs访谈vs二手数据 |
| **理论验证** | 结论与理论假设的一致性 | 假设检验结果 |
| **专家验证** | 专家对结论的认可度 | 德尔菲共识度 |

### 3.2 验证流程

```
Step 1: 独立分析（并行执行）
        ├── 方法A分析 → 结果A
        ├── 方法B分析 → 结果B
        └── 方法C分析 → 结果C

Step 2: 结果对比
        ├── 结论方向是否一致？
        ├── 排序是否一致？
        ├── 关键因素是否重合？
        └── 量化指标（重合度、相关性等）

Step 3: 差异调解
        ├── 若差异>阈值：检查数据质量、方法适用性、专家仲裁
        └── 若差异<阈值：进入结论整合

Step 4: 结论整合
        ├── 共识结论（置信度高）
        ├── 部分共识结论（置信度中）
        └── 差异结论（需说明原因）
```

### 3.3 验证标准

| 研究问题 | 验证方法1 | 验证方法2 | 验证方法3 | 验证标准 |
|----------|-----------|-----------|-----------|----------|
| RQ1 | 描述性统计 | 文本分析 | 案例分析 | 痛点重合度>70% |
| RQ2 | 模糊DEMATEL-ISM | SEM | 贝叶斯网络 | 路径方向一致性>80% |
| RQ3 | 模糊AHP | TOPSIS+VIKOR | 敏感性分析 | 排序相关性ρ>0.7 |
| RQ4 | 技术路线图 | 德尔菲法 | 情景分析 | 趋势方向一致 |
